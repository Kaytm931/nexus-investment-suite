## Skill Overview

Context-engineering is a systematic skill for optimizing Claude Code context-window usage to maximize model effectiveness while minimizing token cost and conversation drift. It implements five core operations: monitor context usage (track token consumption and heavy consumers), reduce context load (evict stale content and minimize loaded files), optimize skill design (progressive disclosure and efficient reference loading), separate planning vs execution (keep execution contexts lean), and apply file-based strategies (externalize large data for up to ~95% token savings). Use it when approaching context limits, building large skills, managing API costs, sustaining multi-turn conversation quality, or integrating with Model Context Protocol servers. Key advantages include 29–39% performance improvements from context editing, drastic token savings, lower latency and costs, and prevention of context overflow and conversation drift.
